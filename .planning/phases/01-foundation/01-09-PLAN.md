  <name>Task 1: Remove unlayered CSS reset from globals.css</name>
  <files>frontend/src/app/globals.css</files>
  <action>
Remove the unlayered `* { box-sizing: border-box; margin: 0; padding: 0; }` block from `frontend/src/app/globals.css` (currently at lines 63-68).

**Why this must be removed:**
Per CSS Cascade Level 5, unlayered author styles always win over ALL `@layer` rules — including `@layer utilities`. In the compiled CSS output, this block appears at byte position ~23,639 (after `@layer utilities` ends at ~991), making it override every Tailwind utility class that sets margin or padding (e.g., `mt-4`, `py-6`, `px-4`, `gap-8`, `space-y-3`, `mb-2`). This causes the unstyled appearance on landing, login, and signup pages.

**Tailwind v4 already provides this reset** inside `@layer base`:
```css
/* Tailwind v4 @layer base includes: */
*, :after, :before, ::backdrop {
  box-sizing: border-box;
  border: 0 solid;
  margin: 0;
  padding: 0;
}
```
The manually authored block is fully redundant AND breaks the cascade.

**Keep everything else in globals.css intact:**
- `@import "tailwindcss"` — keep
- Font-face declaration — keep
- All `:root` CSS custom property tokens — keep
- `.dark` CSS custom property overrides — keep
- `html` and `body` base styles — keep
- `:focus-visible` ring — keep
- `.glass` utility — keep
- `.btn-gradient` utility — keep
- `code, pre, .font-mono` rule — keep

**Only remove** these lines:
```css
/* ── Base Reset ── */
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}
```
(The comment `/* ── Base Reset ── */` should also be removed.)
