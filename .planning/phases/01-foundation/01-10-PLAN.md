**Part A — Add ThemeToggle to `frontend/src/app/page.tsx`:**

Import `ThemeToggle` from `@/components/layout/ThemeToggle` and render it in the public nav bar on the landing page.

**Current nav structure in `frontend/src/app/page.tsx`:**
```tsx
<nav
  className="flex items-center justify-between px-6 h-16 border-b"
  style={{ borderColor: 'var(--color-border)' }}
>

    <Link href="/login" ...>Login</Link>
    <Link href="/signup" ...>Sign Up</Link>
  </div>

```

**Target nav structure** — add ThemeToggle before the Login/Sign Up links:
```tsx
import { ThemeToggle } from '@/components/layout/ThemeToggle'

// In the nav's right-side div:
<div className="flex items-center gap-3">
  <ThemeToggle />
  <Link href="/login" ...>Login</Link>
  <Link href="/signup" ...>Sign Up</Link>
</div>
```

**Implementation steps:**
1. Add `import { ThemeToggle } from '@/components/layout/ThemeToggle'` at the top of `page.tsx` alongside existing imports.
2. In the `<nav>` element, find the `<div className="flex gap-3">` container with the Login and Sign Up links.
3. Add `items-center` to the div's className (already has `flex gap-3` — add `items-center` for vertical alignment with the toggle button).
4. Render `<ThemeToggle />` as the FIRST child inside that div, before the Login link.

**Note:** `ThemeToggle` uses `useTheme` from `next-themes` which requires the `ThemeProvider` already wrapping the whole app in `layout.tsx` — no additional provider needed.

**Part B — Add Playwright test to `e2e/auth.spec.ts`:**

Add a test to the existing `Landing Page` describe block (at the end of that block, before the closing `}`) that asserts the theme toggle button is visible at `/`.

Look at how `ThemeToggle` renders: it uses a `<button>` element. Add a `data-testid="theme-toggle"` attribute if needed (check the actual component). If the ThemeToggle button already has a discernible aria-label (e.g., `aria-label="Toggle theme"`) use that; otherwise use `data-testid`.

**Preferred locator strategy (in priority order):**
1. `page.getByRole('button', { name: /toggle theme/i })` — if ThemeToggle has aria-label="Toggle theme"
2. `page.getByTestId('theme-toggle')` — add `data-testid="theme-toggle"` to ThemeToggle's `<button>` in `frontend/src/components/layout/ThemeToggle.tsx` if no aria-label

Add this test inside the `test.describe('Landing Page', ...)` block:
```typescript
test('theme toggle button is visible in nav', async ({ page }) => {
  await page.goto('/')
  // Use the role+name locator if ThemeToggle has aria-label; otherwise data-testid
  await expect(page.getByRole('button', { name: /toggle theme/i })).toBeVisible()
})
```

If ThemeToggle lacks an accessible label, also update `frontend/src/components/layout/ThemeToggle.tsx` to add `aria-label="Toggle theme"` to the `<button>` element — this satisfies both the Playwright locator and WCAG accessibility.
