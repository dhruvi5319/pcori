## Typography

**Inherited from Phase 1 + Phase 2 — no changes to the 4-size, 2-weight scale:**

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Display | 48px | 600 | 1.1 | Geist Sans | Landing hero only (no change) |
| Heading | 24px | 600 | 1.2 | Geist Sans | Page headers, chart section titles, dialog titles |
| Body | 16px | 400 | 1.5 | Geist Sans | Tooltip text, notification body, description copy |
| Label / Small | 14px | 400 | 1.4 | Geist Sans | Chart axis labels, KPI card sub-labels, notification timestamps |

**Declared sizes: exactly 4 — 14 / 16 / 24 / 48px. No other sizes are part of the type scale.**

**Phase 3 specific usage notes:**

- KPI card primary number: 48px / 600 (Display size — these are the hero metrics)
- KPI card label below number: 14px / 400, muted color (e.g., "Total Plans", "Avg AI Confidence")
- KPI card delta/sub-label (e.g., "+5 today"): 14px / 400 (Label size — green for positive, red for negative, gray for zero)
- Chart section title: 16px / 600 (Body size with weight bump — not 24px; charts are dense)
- Chart axis tick labels: Recharts SVG internal `fontSize` default — not a declared type scale token; executor sets `style={{ fontSize: 12 }}` directly on `<XAxis>`/`<YAxis>` SVG props without adding to the type scale
- Chart custom tooltip card: 14px / 400 (Label size)
- Pipeline log entry: 14px / 400, Geist Mono (Label size in monospace font — log line density handled by compact vertical padding, not reduced font size)
- Pipeline stats header: 16px / 600 (Body size, weight bump for status values)
- Notification item title: 14px / 600 (Label size, weight bump — title is primary scent)
- Notification item body: 14px / 400 (Label size, regular weight)
- Notification timestamp: 14px / 400, muted (Label size — compact enough at 14px with muted color treatment)

**Confidence Distribution Histogram Bar Colors (user decision — locked):**

| Confidence Bucket | Bar Color | Rationale |
|-------------------|-----------|-----------|
| 0.0–0.7 (low) | `#DC2626` red / dark `#EF4444` | Below NEEDS_REVIEW threshold |
| 0.7–0.85 (medium) | `#D97706` amber / dark `#F59E0B` | Review suggested band |
| 0.85–1.0 (high) | `#16A34A` green / dark `#22C55E` | High confidence band |

> Each histogram bar uses the color band matching its bucket, not a single chart color. Buckets straddle the band boundaries (0.7 and 0.85): assign to the lower band if below boundary, upper band at or above.

## Accessibility Contract

**All Phase 1 + Phase 2 WCAG 2.1 AA requirements apply unchanged. Phase 3 additions:**

| Requirement | Implementation |
|-------------|---------------|
| KPI card drag handles | `aria-label="Drag to reorder [metric name] card"` on GripVertical; `role="button"` + keyboard drag support via `@dnd-kit` |
| Drag-and-drop keyboard | `@dnd-kit` provides keyboard-accessible drag via Space to pick up, arrow keys to move, Enter/Space to drop |
| Notification bell | `aria-label="View notifications"` + `aria-haspopup="true"` + `aria-expanded` state |
| Notification count | `aria-live="polite"` on badge element to announce count changes to screen readers |
| Notification drawer | Radix Dialog or custom with focus trap when open; Escape closes; `aria-label="Notifications panel"` |
| Notification item click | Each item: `role="button"` + `tabindex="0"` + `onKeyDown` Enter/Space to activate |
| Notification unread dot | `aria-label="Unread"` on the dot element; it is decorative but its presence communicates state |
| Recharts charts | Each chart section: `aria-label="[Chart title] chart"` on SVG wrapper; custom tooltip via `<title>` element for screen readers |
| Chart empty states | `role="status"` on empty state container for screen reader announcement |
| Switch toggles (preferences) | Radix `Switch` has built-in `role="switch"` + `aria-checked` + keyboard activation (Space) |
| Pipeline state indicator | Colored dot + text label always together (color never sole indicator); `aria-label="Pipeline state: [RUNNING/PAUSED/etc]"` |
| Stage card state | Stage name + state label always explicit text; left border color is supplementary |
| Log level badges | Color + text level always shown together (INFO/WARN/ERROR text label alongside color) |
| Collapsible log panel | `aria-expanded` on toggle button; `aria-controls` pointing to log panel id |
| Analytics loading pulse | `aria-busy="true"` on chart sections when `isLoading === true`; removed when false |
| Date range inputs | Explicit `<label>` for each date input; `aria-describedby` for validation errors |
| KPI card error state retry | `aria-label="Retry loading [metric name]"` on `RotateCcw` icon button | Inline error state on `KpiCard` |
