**ReportService.java**:
- `@Service @Transactional @RequiredArgsConstructor @Slf4j`
- `startGeneration(ExcelGenerateRequest req, UUID ownerId)`: create `ExcelReport` with status=GENERATING; save; submit `@Async` task to call `excelGenerationService.generateExcel()`, store bytes via `fileService` (or to temp file path), update ExcelReport status=READY+filePath; on exception set status=FAILED+errorMessage; return `ExcelReportResponse`
- `downloadReport(UUID reportId)`: find ExcelReport; verify status=READY; read file bytes; return `byte[]` with filename
- `getPreview(String filtersJson, List<String> columns)`: query up to 3 classifications matching filters; return `PreviewResponse(totalCount, 3-row sample)`
- `listReports(UUID ownerId)`: `excelReportRepository.findByConfigurationId...` or all reports for owner
- `createTemplate(ReportConfigRequest req, UUID ownerId)`: check name uniqueness; save `ReportConfiguration`; return `ReportConfigResponse`
- `listTemplates(UUID ownerId)`: return `List<ReportConfigResponse>`
- `updateTemplate(UUID id, ReportConfigRequest req, UUID ownerId)`: find + verify ownership; update; save
- `deleteTemplate(UUID id, UUID ownerId)`: soft-delete via `deletedAt = Instant.now()`
- `runTemplate(UUID templateId, UUID ownerId)`: load template columns/filters; delegate to `startGeneration`

For async Excel generation: use `@Async` on a private `@Transactional(propagation = Propagation.REQUIRES_NEW)` method — same `@Async` pattern as `ClassificationPipeline`. The `classificationExecutor` bean from `AsyncConfig` can be reused or a separate executor defined.
