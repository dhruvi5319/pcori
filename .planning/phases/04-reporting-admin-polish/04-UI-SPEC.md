Only Phase 4 additions and overrides are declared here.

## Typography

**Inherited from Phases 1–3 — no changes to the 4-size, 2-weight scale:**

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Display | 48px | 600 | 1.1 | Geist Sans | Landing hero only (no change) |
| Heading | 24px | 600 | 1.2 | Geist Sans | Page headers, dialog titles, help article titles |
| Body | 16px | 400 | 1.5 | Geist Sans | Form labels, descriptions, help article body prose |
| Label / Small | 14px | 400 | 1.4 | Geist Sans | Table cell text, column checkboxes, role chip text, timestamps |

**Declared sizes: exactly 4 — 14 / 16 / 24 / 48px. No other sizes are part of the type scale.**

**Phase 4 specific usage notes:**

- Report page section tabs ("My Reports" / "Ad-hoc Builder" / "Templates"): 16px / 400 tab label; active tab gets `border-b-2 border-accent` underline
- Column selector checkbox label: 14px / 400
- Preview row count callout ("143 matching rows"): 16px / 600 (body size + weight bump — it's the decision point)
- Large-report warning: 14px / 400, amber color — same pattern as UrgentActionBanner (Phase 3)
- User table: Username 14px / 600 (Geist Mono, identifying field), all other columns 14px / 400
- Role chip inside table: 14px / 400 — same as status badge label size (Phase 2)
- Help article Markdown prose: 16px / 400, line-height 1.6 (slightly looser than body default for readability of longer text; NOT a new type scale token — set via `prose` CSS class on the article container)
- FAQ accordion question: 16px / 400 (body weight, not semibold — question is a trigger, not a heading)
- FAQ accordion answer: 14px / 400 (label size — tighter density inside accordion panel)
- Feedback widget "Was this helpful?" label: 16px / 400 (body size — it's a conversational prompt)

- Closed item trigger: 16px / 400, `text-gray-900 dark:text-gray-100`; ChevronRight icon rotates to ChevronDown on open
- Open item trigger: same text color; accent blue left border `border-l-2 border-accent` on the open panel
- Answer panel bg: secondary bg `#F4F6F9 dark:#141414`

**Article feedback widget states:**
- Default (no selection): ThumbsUp + ThumbsDown buttons — gray icon `text-gray-400`, no fill

- H2 headings inside Markdown: 16px / 600 (body size + weight bump — keeps article compact)
- Code blocks inside Markdown: Geist Mono 14px; `bg-secondary` bg; `rounded-md p-4`

**FeedbackWidget spec:**
- Default: "Was this helpful?" (16px/400) + [👍 Yes] [👎 No] side by side
- ThumbsUp/ThumbsDown buttons: 44px × 44px touch target; icon 20px lucide
- On click: optional `<textarea>` expands below (max-height 96px, resize-none) with "Add a comment (optional)" placeholder
- Auto-submit on button click if no comment; shows [Submit Feedback] button when textarea is visible
- Post-submit: entire widget replaced by `CheckCircle2` (green, 20px) + "Thank you for your feedback!" (14px/400 muted)
- Widget is shown once per article visit; if user already submitted this session, shows the submitted state immediately

**FAQ Accordion spec:**
- Radix `Accordion` with `type="single"` (one open at a time per group) + `collapsible`
- Each FAQ item: trigger = question text (16px/400) + ChevronRight icon right-aligned; ChevronRight rotates 90° on open (CSS transition 0.2s ease)
- Open item: `border-l-2 border-[#1D4ED8]` accent left border on the content panel
- Answer panel: `bg-[#F4F6F9] dark:bg-[#141414]` secondary bg; `padding: 16px` (md token); answer text 14px/400
- Category filter dropdown: Radix `Select`; "All" shows all categories; selecting one scrolls to that group + hides others

Radix Accordion provides `data-state="open"/"closed"` on the content element. Use `data-state` to conditionally apply accent left border in Tailwind: `data-[state=open]:border-l-2 data-[state=open]:border-[#1D4ED8]`.

## Accessibility Contract

**All Phases 1–3 WCAG 2.1 AA requirements apply unchanged. Phase 4 additions:**

| Requirement | Implementation |
|-------------|---------------|
| Column selector checkboxes | Radix `Checkbox` has built-in `role="checkbox"` + `aria-checked`; each has visible label text |
| Role checkboxes in user dialogs | Same Radix Checkbox pattern; group wrapped in `role="group"` with `aria-labelledby="role-group-label"` |
| FAQ accordion | Radix `Accordion` provides `role="button"` on triggers, `aria-expanded`, `aria-controls`; keyboard: Space/Enter to open |
| Article feedback thumbs | `ThumbsUp`/`ThumbsDown` buttons: `aria-label="This article was helpful"` / `aria-label="This article was not helpful"` |
| Feedback submit | `aria-label="Submit article feedback"` on the submit button |
| Report status cells | Status text is always explicit (never color-only); "Generating…", "Ready", "Failed" text accompanies the dot |
| Template/filter name inputs | Explicit `<label>` + `aria-describedby` pointing to inline error element |
| Deactivate confirmation | Dialog trap focus; dismiss button auto-focused (safer default for destructive action) |
| Help article Markdown | Rendered Markdown does NOT inject `<script>` or raw HTML; sanitized |
| Search results overlay | `role="listbox"` on results container; each result is `role="option"` |
| Role chips in table | `aria-label="Roles: [comma-separated list]"` on the cell; chips are decorative within |
| `UserRowActions` icon buttons | `aria-label="View user [username]"` on Eye icon button; `aria-label="Edit user [username]"` on Edit icon button; `aria-label="Deactivate [username]"` (or "Reactivate [username]") on the action button |
| `···` MoreHorizontal trigger in Users table | `aria-label="Actions for [username]"` on the MoreHorizontal icon button that opens the DropdownMenu |
